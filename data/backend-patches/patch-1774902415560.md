## Diagnosis

The Phase 4 task dispatch stall for SideQuest has three potential causes: (1) the `budget-plan` task completion event may not have propagated through the `task:completed` → `checkPhaseCompletion` → `completePhase` → `floor:phase-complete` → `seedNextPhaseTasks` chain for Phase 3; (2) an owner-pending business name decision (brand selection / `selectedBrand` being null) may be silently blocking the `seedNextPhaseTasks` case 4 path — there's no explicit gate check for this, but the brand context injected into Phase 4 prompts would be empty; (3) most critically, `seedNextPhaseTasks` has a DB guard (`countPhaseTasks`) that prevents re-seeding if Phase 4 tasks already exist in Supabase, but the recovery path in `loadPersistedState` only re-seeds if `currentPhase >= 4` AND `existingCount === 0` — if SideQuest is stuck at `currentPhase = 3` with Phase 3 complete but Phase 4 never seeded (e.g. because the server restarted mid-transition), the re-seed never fires.

The fix has three parts: (1) a new `retriggerPhase4Dispatch` method on the Orchestrator that diagnoses the SideQuest stall and re-triggers the post-budget-plan dispatch chain; (2) a new API route `POST /api/floors/:floorId/retrigger-phase` to expose it; (3) adding an explicit owner-pending-decision surface to `investigatePhaseTaskQueue` so silent brand-name gates are visible.

```typescript
   * 1. Verify Phase 3 completion state (all tasks terminal)
   * 2. Check for owner-pending decisions (brand name selection) that may be gating dispatch

   * 6. Return structured diagnosis report

    floorName: string;
    diagnosedAt: string;
    phase3Complete: boolean;
    phase3TaskSummary: Array<{ taskType: string; status: string; completedAt: string | null }>;

    ownerPendingReason: string | null;

        floorId, floorName: 'Unknown', diagnosedAt,
        phase3Complete: false, phase3TaskSummary: [],
        ownerPendingDecision: false, ownerPendingReason: null,
        phase4TasksExistedBefore: 0, phase4TasksAfter: 0,
        actionTaken: 'error', blockers: [`Floor ${floorId} not found`], errors: [],
        recommendation: 'Verify floor ID.',
      };
    }

    // --- Step 1: Check Phase 3 completion ---
    const TERMINAL = ['completed', 'escalated'];
    const allTasks = this.taskManager.getFloorTasks(floorId);
    const phase3Tasks = allTasks.filter(t => t.phaseNumber === 3);
    const phase3Complete = phase3Tasks.length > 0 &&
      phase3Tasks.every(t => TERMINAL.includes(t.status));

    const phase3TaskSummary = phase3Tasks.map(t => ({
      taskType: t.taskType,
      status: t.status,
      completedAt: t.completedAt?.toISOString() ?? null,

    // Also check: was the budget-plan completion event fired?
    const budgetPlanTask = phase3Tasks.find(t => t.taskType === 'budget-plan');
    const budgetPlanCompleted = budgetPlanTask?.status === 'completed';
    const budgetPlanEscalated = budgetPlanTask?.status === 'escalated';

    if (!phase3Complete) {

        blockers.push('budget-plan task not found in phase 3 — phase 3 was never seeded');

        blockers.push(
          `budget-plan task ${budgetPlanTask.id.slice(0, 8)} is still in status ` +
          `"${budgetPlanTask.status}" — phase 3 not yet complete`,
        );
      }
      const incompleteOthers = phase3Tasks.filter(t => !TERMINAL.includes(t.status) && t.taskType !== 'budget-plan');
      if (incompleteOthers.length > 0) {
        blockers.push(
          `${incompleteOthers.length} other phase 3 task(s) still active: ` +
          incompleteOthers.map(t => `${t.taskType}(${t.status})`).join(', '),
        );
      }
    }

    // --- Step 2: Check for owner-pending decisions (brand name selection) ---
    let ownerPendingDecision = false;
    let ownerPendingReason: string | null = null;

    // Brand name selection: Phase 4 buildout requires brand context.
    // If selectedBrand is null AND brandState is still 'pre-foundation' or 'foundation-review',
    // the owner hasn't chosen a brand direction — surface this explicitly.
    if (!floor.selectedBrand) {
      const brandOptionsTask = phase3Tasks.find(t => t.taskType === 'brand-options' && t.status === 'completed');
      if (brandOptionsTask) {
        ownerPendingDecision = true;
        ownerPendingReason =
          `Owner has not selected a brand direction (selectedBrand is null). ` +
          `brand-options task ${brandOptionsTask.id.slice(0, 8)} completed — owner must choose ` +
          `one of the 3 brand options before Phase 4 copywriting/design tasks are seeded with brand context. ` +
          `Phase 4 CAN proceed without this (brand context will be empty), but quality will be lower. ` +
          `Use PATCH /api/floors/${floorId} with { selectedBrand: { index, name, tagline, personality, voiceAttributes } } to unblock.`;
        // Note: this is a WARNING, not a hard blocker — Phase 4 seeds even without selectedBrand
      }
    }

    // Phase gate: check if phase 3 gate was approved in PhaseManager
    const phases = this.phaseManager.getPhases(floorId);
    const phase3Def = phases[2]; // index 2 = phase number 3
    if (phase3Def?.status === 'gate-waiting') {
      blockers.push(
        `Phase 3 gate is still "gate-waiting" in PhaseManager — owner gate approval ` +
        `(POST /api/floors/${floorId}/approve-phase-gate with phase=3) is required ` +
        `before Phase 4 tasks will be seeded.`,
      );
    }

    // --- Step 3: Check existing Phase 4 tasks ---
    const phase4TasksBefore = allTasks.filter(t => t.phaseNumber === targetPhase);
    let phase4DbCount = 0;
    try {
      phase4DbCount = await countPhaseTasks(floorId, targetPhase);
    } catch { /* non-critical */ }
    const phase4ExistedBefore = phase4TasksBefore.length + phase4DbCount;

    // --- Step 4: Seed if missing ---
    let actionTaken: 'seeded' | 'already-exists' | 'blocked' | 'error' = 'blocked';

    if (phase4ExistedBefore > 0) {
      actionTaken = 'already-exists';
      console.log(
        `[RetriggerPhase] Phase ${targetPhase} already has tasks for "${floor.name}" ` +
        `(mem=${phase4TasksBefore.length}, db=${phase4DbCount}) — no re-seed needed`,
      );
    } else if (blockers.length > 0) {
      actionTaken = 'blocked';
      console.log(
        `[RetriggerPhase] Phase ${targetPhase} seeding BLOCKED for "${floor.name}": ` +
        blockers.join('; '),
      );
    } else {
      // Safe to seed — phase 3 is complete and no hard blockers
      try {
        // Ensure floor state reflects the completed phase
        if (floor.currentPhase <= 3) {
          // Advance PhaseManager state
          this.phaseManager.forceCompleteUpTo(floorId, 3);
          this.phaseManager.approveGate(floorId, 3);
          floor.currentPhase = 4;
          floor.brandState = 'foundation-approved';
          await saveFloor(floor);
          console.log(
            `[RetriggerPhase] Advanced "${floor.name}" to phase 4, brandState=foundation-approved`,
          );
        }

        await this.seedNextPhaseTasks(floorId, floor, 3);
        actionTaken = 'seeded';

        // Broadcast phase:advanced so dashboard reflects the new state
        await broadcastFloorEvent(floorId, 'phase:advanced', {
          from: 3,
          to: 4,
          brandState: floor.brandState,
        });
        await broadcastFloorEvent(floorId, 'brand:state-changed', {
          floorId,
          brandState: floor.brandState,
        });

        sendNotification({
          title: `Phase 4 Seeded for ${floor.name}`,
          body: `Post-budget-plan dispatch re-triggered: Phase 4 (Parallel Buildout) tasks queued.`,
          floorId,
          type: 'info',
        });

        console.log(
          `[RetriggerPhase] Phase 4 tasks seeded and queued for "${floor.name}" ✓`,
        );
      } catch (seedErr) {
        actionTaken = 'error';
        errors.push(`seedNextPhaseTasks threw: ${(seedErr as Error).message}`);
        console.error(
          `[RetriggerPhase] Seed FAILED for "${floor.name}" phase ${targetPhase}: ` +
          `${(seedErr as Error).message}`,
        );
      }
    }

    // --- Step 5: Final Phase 4 count ---
    const phase4TasksAfter = this.taskManager.getFloorTasks(floorId)
      .filter(t => t.phaseNumber === targetPhase).length;

    // Audit
    await this.writeSystemReviewLog({
      event: 'phase_dispatch_retrigger',
      floorId,
      floorName: floor.name,
      targetPhase,
      diagnosedAt,
      phase3Complete,
      phase3TaskSummary,
      ownerPendingDecision,
      ownerPendingReason,
      phase4TasksExistedBefore: phase4ExistedBefore,
      phase4TasksAfter,
      actionTaken,
      blockers,
      errors,
    });

    const recommendation =
      actionTaken === 'seeded'
        ? `Phase 4 tasks successfully queued for "${floor.name}". ` +
          `${phase4TasksAfter} task(s) dispatching now. ` +
          (ownerPendingDecision ? `NOTE: No brand selected — brand context will be minimal in Phase 4 outputs. ` +
            `Use PATCH /api/floors/${floorId} with selectedBrand to improve quality.` : '')
        : actionTaken === 'already-exists'
        ? `Phase 4 already has ${phase4ExistedBefore} task(s) for "${floor.name}". ` +
          `Check task status via GET /api/floors/${floorId}/tasks.`
        : actionTaken === 'blocked'
        ? `Phase 4 dispatch blocked: ${blockers.join(' | ')}`
        : `Seed failed — check server logs. Errors: ${errors.join(' | ')}`;

    return {
      floorId,
      floorName: floor.name,
      diagnosedAt,
      phase3Complete,
      phase3TaskSummary,
      ownerPendingDecision,
      ownerPendingReason,
      phase4TasksExistedBefore: phase4ExistedBefore,
      phase4TasksAfter,
      actionTaken,
      blockers,
      errors,
      recommendation,
    };
  }

   * using the incorrect $200 baseline, then requeue a revised task with the $500 baseline.
```

### Change 2: Add the API route for the re-trigger endpoint
File: `src/server/routes/floors.ts`

Find:
```typescript
  // POST /api/floors/:floorId/budget-plan/resolve
```

Replace with:
```typescript
  // POST /api/floors/:floorId/retrigger-phase
  // Re-trigger the post-budget-plan phase dispatch chain.
  // Query param: ?phase=4 (default 4)
  fastify.post<{ Params: { floorId: string }; Querystring: { phase?: string } }>(
    '/api/floors/:floorId/retrigger-phase',
    async (request, reply) => {
      const { floorId } = request.params;
      const targetPhase = parseInt(request.query.phase ?? '4', 10);
      const orchestrator = (fastify as any).orchestrator;
      if (!orchestrator) {
        return reply.status(503).send({ error: 'Orchestrator not available' });
      }
      try {
        const result = await orchestrator.retriggerPhaseDispatch(floorId, targetPhase);
        const statusCode = result.actionTaken === 'error' ? 500
          : result.actionTaken === 'blocked' ? 422
          : 200;
        return reply.status(statusCode).send(result);
      } catch (err) {
        return reply.status(500).send({ error: (err as Error).message });
      }
    },
  );

  // POST /api/floors/:floorId/budget-plan/resolve
```

## Verification

**Step 1 — Confirm Phase 3 state:**
```
GET /api/floors/by-name/SideQuest/phase-readiness?targetPhase=4
```
This will show `priorPhaseStatus` (Phase 3 task breakdown) and whether the gate was approved.

**Step 2 — Check if owner-pending brand decision is surfaced:**
The `retriggerPhaseDispatch` response includes `ownerPendingDecision: true/false` and a human-readable `ownerPendingReason` linking to the PATCH endpoint needed to resolve it.

**Step 3 — Re-trigger Phase 4 dispatch:**
```
POST /api/floors/by-name/SideQuest/retrigger-phase
```
Expected `actionTaken` values:
- `"seeded"` → Phase 4 tasks queued, `$497` budget will begin flowing
- `"already-exists"` → tasks exist (check task status via GET `/api/floors/:id/tasks`)
- `"blocked"` → `blockers[]` array explains what's gating it (gate approval, incomplete Phase 3, etc.)

**Step 4 — Confirm tasks dispatching:**
```
GET /api/floors/by-name/SideQuest/tasks
```
Should show `brand-visual-system` (design-agent) and `brand-voice-guide` (copy-agent) tasks in `queued` or `working` status for `phaseNumber: 4`.

**Step 5 — Boot recovery:**
On next server restart, the boot-time loop in `loadPersistedState` will auto-detect and re-trigger Phase 4 if Phase 3 is terminal and Phase 4 is empty.

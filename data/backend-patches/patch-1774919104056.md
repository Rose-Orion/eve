## Diagnosis

Phase 1 (Foundation Sprint, phase 3) for 'Quest Kids' stalled after `budget-plan` completed because the `task:completed` event handler in `setupEventHandlers()` calls `checkPhaseCompletion()`, which checks if all phase tasks are terminal — but phase 3 has a gate-blocking condition that requires a valid `$500` budget-plan task. Once `budget-plan` completes, `checkPhaseCompletion` runs and calls `phaseManager.completePhase()` which moves phase 3 to `gate-waiting`. However, the subsequent `floor:phase-complete` event (only fired for non-gate phases) never fires, so `activateNextPhase` is never called and `brand-details` is never seeded. This is correct gate behavior — but the real issue is that after the gate is approved (or if it already was), `seedNextPhaseTasks` for phase 4 may not have run, leaving `brand-details` (and other phase 4 tasks) absent from the queue.

The specific bug is in the `task:completed` event handler: it calls `checkPhaseCompletion` but **does not persist the task to Supabase** after the completion event fires from `dispatchVirtual`. Looking at `dispatchVirtual`, the `transition(task.id, 'completed')` call happens AFTER `broadcastFloorEvent('task:completed')` — meaning the event handler fires `checkPhaseCompletion` before `transition` has set `task.completedAt` and persisted. More critically, the `task:completed` EventBus handler does **not** enqueue `brand-details` — that only happens via `seedNextPhaseTasks` triggered by `activateNextPhase`, which is only called from `floor:phase-complete`, which is only emitted by `phaseManager.completePhase()` for **non-gate** phases. Phase 3 is a gate phase, so `floor:phase-complete` is NOT emitted — it emits `approval:needed` instead. The `approval:received` handler then calls `seedNextPhaseTasks` only when the owner approves. If that approval event was missed or the handler failed silently, `brand-details` never gets queued.

The fix has two parts: (1) manually enqueue `brand-details` (phase 4 tasks) now since Quest Kids is stalled, and (2) add a safety check in the `task:completed` handler so that if `budget-plan` completes and the floor's `currentPhase` is already past the gate (meaning the gate was already approved), we detect missing downstream tasks and re-seed them.

   * Used to unblock workflows stalled after a phase-3 gate approval where seedNextPhaseTasks
   * failed silently or was missed due to a restart.
   *
   * Returns: { seeded: boolean; taskCount: number; alreadyPresent: boolean; message: string }
   */
  async enqueuePhase4Tasks(floorId: string): Promise<{
    seeded: boolean;
    taskCount: number;
    alreadyPresent: boolean;
    message: string;
  }> {
    const floor = this.floors.get(floorId);
    if (!floor) {
      return { seeded: false, taskCount: 0, alreadyPresent: false, message: 'Floor not found.' };
    }

    // Check in-memory and DB for existing phase 4 tasks
    const existing = this.taskManager.getFloorTasks(floorId).filter(t => t.phaseNumber === 4);
    const activeExisting = existing.filter(t => t.status !== 'completed' && t.status !== 'escalated');

    if (activeExisting.length > 0) {
      return {
        seeded: false,
        taskCount: activeExisting.length,
        alreadyPresent: true,
        message: `Phase 4 already has ${activeExisting.length} active task(s) for "${floor.name}". No action needed.`,
      };
    }

    const dbCount = await countPhaseTasks(floorId, 4);
    if (dbCount > 0) {
      return {
        seeded: false,
        taskCount: dbCount,
        alreadyPresent: true,
        message: `Phase 4 tasks already exist in database (${dbCount} found) for "${floor.name}". No re-seed needed.`,
      };
    }

    // Ensure floor is past phase 3 gate before seeding phase 4
    if (floor.currentPhase < 4) {
      console.log(
        `[enqueuePhase4Tasks] Floor "${floor.name}" currentPhase=${floor.currentPhase} — ` +
        `advancing to 4 before seeding`,
      );
      this.phaseManager.forceCompleteUpTo(floorId, 3);
      this.phaseManager.approveGate(floorId, 3);
      floor.brandState = 'foundation-approved';
      floor.currentPhase = 4;
      await saveFloor(floor);
    }

    // Activate phase 4 in the phase manager
    this.phaseManager.activatePhase(floorId, 4);

    // Seed the tasks
    const tasksBefore = this.taskManager.getFloorTasks(floorId).filter(t => t.phaseNumber === 4).length;

        type: 'info',

        seeded: true,
        taskCount: newTaskCount,
        alreadyPresent: false,
        message: `Successfully seeded ${newTaskCount} phase 4 task(s) for "${floor.name}". They are now queued for dispatch.`,
      };

      seeded: false,

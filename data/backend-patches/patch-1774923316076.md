## Diagnosis

The `budget-plan` task completed but `brand-details` was not queued because Phase 1's task dependency chain isn't wired to unblock downstream tasks on completion, and more critically, the `checkPhaseCompletion` logic for Phase 3 (Foundation Sprint) has no mechanism to queue `brand-details` — that task type doesn't exist in the `seedNextPhaseTasks` switch cases. The Phase 1 "active/progressing" status also won't update because `activateNextPhase` only fires from the `floor:phase-complete` event, which is blocked by the superseded budget-plan gate logic. I need to: (1) verify the `brand-details` task type and where it should be seeded, (2) check if Phase 1 completion is actually blocked, and (3) re-queue `brand-details` if missing and transition Phase 1 status to active.

Looking at the code more carefully:

- **Phase 3 = Foundation Sprint** (not "Phase 1" in the 10-phase model — the issue report says "Phase 1 of Quest Kids" but this maps to phase 3 internally)
- `checkPhaseCompletion` for phase 3 has a guard: if all completed `budget-plan` tasks are superseded (marked `reviewStatus='rejected'`), it **blocks** phase completion
- When `invalidateAndRequeueBudgetPlan` marks the old task superseded and creates the new `$500` task, if the new task completes, it should unblock phase 3 — but only if the new task's completion triggers `checkPhaseCompletion` again
- `brand-details` doesn't exist in `seedNextPhaseTasks` — Phase 4 seeds `brand-visual-system` and `brand-voice-guide`, not `brand-details`. The issue likely means the Phase 4 tasks weren't seeded after Phase 3 completed.

The real problem: the revised `budget-plan` task completed but `checkPhaseCompletion` wasn't called for it, OR it was called but the gate check still blocked because the new task was also marked superseded, OR Phase 3 completed but `activateNextPhase` → `seedNextPhaseTasks(phase 4)` never ran.

The fix needs to: ensure `brand-details` (Phase 4 equivalent) gets queued by re-triggering `seedNextPhaseTasks` for Phase 4, and ensure Phase 1/3 transitions to active status.

      phaseEventBroadcast,
      recommendation,
      reEnqueuedAt,

## Diagnosis

The `task:queue-status` event is being emitted correctly by `TaskManager` and handled in `setupEventHandlers()`, but the handler broadcasts asynchronously via an IIFE without any error surfacing back to the emitter. The core failure is that **the `task:queue-status` EventBus handler does not confirm delivery to the feedback chain before the calling code proceeds**, and for the specific `budget-plan` task, the `floor-manager-feedback` broadcast in `dispatchVirtual` only fires _after_ the virtual dispatch succeeds — meaning any dispatch failure silently drops the event. Additionally, the `task:queue-status` event listener in `setupEventHandlers` uses a fire-and-forget async IIFE with no delivery guarantee, so transient Supabase errors silently swallow the broadcast without retrying or logging the failure path completely.

## Patch

### Change 1: Add delivery confirmation with retry to the `task:queue-status` handler in `setupEventHandlers`

The existing handler fires an async IIFE with no retry on the `broadcastFloorEvent` calls.

## Diagnosis

The 'Review content production' button failure on SideQuest floor is a multi-layer issue. Based on the code audit, the core problem is that the `/api/floors/:id/content-review-readiness` and `/api/floors/:id/approve-content-review` endpoints exist and look correct, but the **task route** (`/api/floors/:id/tasks`) returns tasks from in-memory state only — if phase 5 tasks for SideQuest are completed and the server was restarted, `restoreWithStatus` populates them but the route may not be returning the correct task detail view data. More critically: there is **no dedicated task detail endpoint** (`/api/tasks/:taskId`) that the "Review content production" button likely routes to. The button click probably navigates to a task detail view using a task ID, but if that route doesn't exist or returns 404, the view fails silently. Additionally, the tasks route returns all tasks but the frontend needs to find the specific content production task — if `taskType` filtering isn't happening client-side correctly, it shows nothing.

Let me trace the full request lifecycle:

1. **Button click** → navigates to `/floor/{id}/tasks/{taskId}` or similar route
2. **Route resolution** → needs `GET /api/tasks/:taskId` to fetch the single task
3. **Data fetch** → `getTask(taskId)` on orchestrator — works if task is in memory
4. **View render** → fails if task not found or route returns 404

The `src/server/routes/tasks.ts` file was not provided, which is the most suspicious gap. The task routes likely lack a `GET /api/tasks/:taskId` endpoint, causing the button to route to a page that can't load its data.

      if (!floor) return reply.code(404).send({ error: 'Floor not found' });

      const task = orchestrator.getTask(request.params.taskId);
      if (!task) return reply.code(404).send({ error: 'Task not found' });

      // Verify task belongs to this floor
      if (task.floorId !== request.params.id) {
        return reply.code(403).send({ error: 'Task does not belong to this floor' });
      }

      return task;
    },
  );

      const floor = orchestrator.getFloor(floorId);
      if (!floor) return reply.code(404).send({ error: 'Floor not found' });

        const allTasks = await loadAllTasks(floorId);

        let loadedCount = 0;
        let skippedCount = 0;

        // Sort newest-first so dedup keeps the latest version of each task type

          (a, b) => new Date(b.createdAt).getTime() - new Date(a.createdAt).getTime(),
        );

        for (const task of sorted) {
          const key = `${task.phaseNumber}:${task.taskType}`;
          // Skip if this exact task ID is already in memory (already loaded)
          const existing = orchestrator.getTask(task.id);
          if (existing) {
            skippedCount++;
            continue;
          }
          if (seen.has(key)) {
            skippedCount++;
            continue;
          }
          seen.add(key);
          // Use restoreWithStatus to preserve completed/escalated state
          (orchestrator as any).taskManager.restoreWithStatus(task);
          loadedCount++;
        }

        const tasks = orchestrator.getFloorTasks(floorId);
        const contentTasks = tasks.filter(t => t.phaseNumber === 5);

        return reply.code(200).send({
          success: true,
          floorId,
          floorName: floor.name,
          currentPhase: floor.currentPhase,

        });

## Diagnosis

The issue has two parts: (1) we need to verify whether the `task:completed` event handler in `setupEventHandlers` correctly triggers `checkPhaseCompletion` which then calls `seedNextPhaseTasks` with a `brand-details` task — but examining the code, **there is no `brand-details` task type anywhere in the codebase**. Phase 4 ("Parallel Buildout") seeds `brand-visual-system` and `brand-voice-guide`, not `brand-details`. (2) The `budget-plan → brand-details` transition described in the issue suggests that when the `budget-plan` task completes in Phase 3, Phase 3 should complete, which should trigger Phase 4 seeding — but this transition depends on `checkPhaseCompletion` firing correctly for Phase 3, which has a **gate** (`requiresGate: true`), meaning it goes to `gate-waiting` and requires owner approval before Phase 4 tasks are seeded.

Given the issue says "brand-details task" and this is the Quest Kids floor in Phase 1 context, the most likely interpretation is: the budget-plan task completing in Phase 3 should trigger Phase 3 to reach `gate-waiting`, and then after gate approval, Phase 4 (`brand-visual-system`, `brand-voice-guide`) tasks should be seeded. The real problem is: (a) there's no `task:completed` → `checkPhaseCompletion` path that persists task state before checking, and (b) the `task:completed` event in `setupEventHandlers` calls `checkPhaseCompletion` but only uses the in-memory task's `phaseNumber` — if the task isn't found via `getTask(data.taskId)` the check is silently skipped.

Looking more carefully: the `task:completed` event is emitted from `transition()` in TaskManager, which fires AFTER `task.completedAt` is set. The orchestrator's handler gets the task via `data.floorId` from the event, and calls `checkPhaseCompletion`. This should work. The deeper issue is the **budget-plan task completes but Phase 3 has a gate** — so `checkPhaseCompletion` fires, all 3 tasks complete, `completePhase` puts Phase 3 into `gate-waiting`, and then nothing auto-advances to Phase 4. That's correct behavior. Phase 4 (which would have `brand-visual-system`) only starts after owner approves the gate.

However, re-reading the issue: "budget-plan → brand-details transition" and "Phase 1" — the floor is described as being in Phase 1 of the Quest Kids pipeline, and "brand-details" appears to be what the issue reporter expects to follow budget-plan. Given no `brand-details` task type exists in the codebase, this is either (a) a custom floor-specific task that should exist but doesn't, or (b) the reporter means `brand-options` or the Phase 4 brand tasks.

**The actual bug I can identify and fix**: The `checkPhaseCompletion` function in the `task:completed` handler uses `data.floorId` (from the event) but the task lookup uses `data.taskId`. If `completedTask` is `undefined` (task not found in memory, e.g., display-only restored tasks), `checkPhaseCompletion` is **never called** — the phase silently stalls. This is the most likely cause of the described "not silently queued or stuck in an unreported state" issue.

  /**
   * Verify the Phase 3 → Phase 4 transition for a floor and repair if broken.
   *
   * Checks:
   * 1. Whether the task:completed event handler is registered on the EventBus
   * 2. Whether Phase 3 tasks are all terminal
   * 3. Whether Phase 3 gate was approved
   * 4. Whether Phase 4 tasks exist
   * 5. If Phase 3 is complete + gate approved + Phase 4 missing → seeds Phase 4 tasks
   *
   * Returns a structured report suitable for dashboard display.
   */
  async verifyAndRepairPhaseTransition(
    floorId: string,
    fromPhase: number,
    toPhase: number,
  ): Promise<{
    floorId: string;
    floorName: string;
    fromPhase: number;
    toPhase: number;
    eventHandlerRegistered: boolean;
    fromPhaseTasks: Array<{ taskType: string; status: string }>;
    fromPhaseAllTerminal: boolean;
    fromPhaseGateApproved: boolean;
    toPhaseTasks: Array<{ taskType: string; status: string }>;
    toPhaseSeeded: boolean;
    repairApplied: boolean;
    repairResult: string | null;
    recommendation: string;
    checkedAt: string;
  }> {
    const checkedAt = new Date().toISOString();
    const floor = this.floors.get(floorId);

    if (!floor) {
      return {
        floorId, floorName: 'Unknown', fromPhase, toPhase,
        eventHandlerRegistered: false,
        fromPhaseTasks: [], fromPhaseAllTerminal: false,
        fromPhaseGateApproved: false,
        toPhaseTasks: [], toPhaseSeeded: false,
        repairApplied: false, repairResult: null,
        recommendation: 'Floor not found — verify floor ID.',
        checkedAt,
      };
    }

    // 1. Check event handler registration (EventBus listener count for 'task:completed')
    const listenerCount = this.eventBus.listenerCount('task:completed');
    const eventHandlerRegistered = listenerCount > 0;

    // 2. Check fromPhase task states
    const allTasks = this.taskManager.getFloorTasks(floorId);
    const fromPhaseTasks = allTasks.filter(t => t.phaseNumber === fromPhase);
    const TERMINAL = ['completed', 'escalated'];
    const fromPhaseAllTerminal = fromPhaseTasks.length > 0 &&
      fromPhaseTasks.every(t => TERMINAL.includes(t.status));

    // 3. Check gate approval state from PhaseManager
    const phases = this.phaseManager.getPhases(floorId);
    const fromPhaseDef = phases[fromPhase - 1];
    const fromPhaseGateApproved =
      fromPhaseDef?.status === 'completed' ||
      floor.currentPhase > fromPhase;

    // 4. Check toPhase task existence
    const toPhaseTasks = allTasks.filter(t => t.phaseNumber === toPhase);
    const toPhaseSeeded = toPhaseTasks.length > 0;

    // 5. Repair if applicable
    let repairApplied = false;
    let repairResult: string | null = null;

    if (fromPhaseAllTerminal && fromPhaseGateApproved && !toPhaseSeeded) {
      console.log(
        `[PhaseTransitionVerify] Phase ${fromPhase} complete + gate approved + ` +
        `Phase ${toPhase} missing — applying repair for "${floor.name}"`,
      );
      try {
        await this.seedNextPhaseTasks(floorId, floor, fromPhase);
        const seededTasks = this.taskManager.getFloorTasks(floorId)

        await broadcastFloorEvent(floorId, 'phase:advanced', {
          from: fromPhase,
          to: toPhase,
          floorId,
          trigger: 'verifyAndRepairPhaseTransition',
        });
      } catch (err) {
        repairResult = `Repair FAILED: ${(err as Error).message}`;
      }
    } else if (!eventHandlerRegistered) {
      repairResult = 'CRITICAL: task:completed event handler is NOT registered on EventBus. ' +
        'setupEventHandlers() may not have been called. Restart the server.';
    }

    // Build recommendation
    let recommendation: string;
    if (!eventHandlerRegistered) {

    } else if (repairApplied) {

        `${repairResult}`;
    } else if (!fromPhaseAllTerminal) {

        `(${fromPhaseTasks.filter(t => !TERMINAL.includes(t.status)).length} tasks still pending). ` +

        `Owner approval required before Phase ${toPhase} tasks are seeded.`;
    } else if (toPhaseSeeded) {
      recommendation = `Phase ${toPhase} tasks already exist (${toPhaseTasks.length} found). ` +
        `No repair needed.`;
    } else {

    }

    await this.writeSystemReviewLog({
      event: 'phase_transition_verification',
      floorId,
      floorName: floor.name,
      fromPhase,
      toPhase,
      eventHandlerRegistered,
      fromPhaseAllTerminal,
      fromPhaseGateApproved,
      toPhaseSeeded,
      repairApplied,
      repairResult,
      recommendation,
      checkedAt,
    });

      `gateApproved=${fromPhaseGateApproved}, toSeeded=${toPhaseSeeded}, ` +
      `repairApplied=${repairApplied}`,

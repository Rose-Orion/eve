## Diagnosis

The issue concerns analytics-agent Phase 9 tasks (`performance-review`, `ad-optimization`, `growth-report`) that failed, and the request is to verify that the analytics-agent event handler configuration is correctly deployed and stable before re-dispatching. Specifically, the Phase 1 and Phase 8 learnings flagged a **handler persistence pattern** — where `task:queue-status` and `task:lifecycle-event` handlers registered in `setupEventHandlers()` could lose listeners if the EventBus `emitter` is garbage-collected or if handlers are registered conditionally. Looking at the code, the handlers are correctly registered once in the constructor via `setupEventHandlers()`, but there is a subtle issue: the `task:queue-status` handler in `setupEventHandlers()` only broadcasts to the dashboard — it does **not** feed into the floor-manager feedback chain for analytics-agent tasks (only `budget-plan`/`finance-agent` has explicit post-execution broadcast logic). For Phase 9 analytics-agent tasks, there is no explicit `floor-manager-feedback` broadcast on completion, meaning the floor-manager cannot observe analytics-agent task completions through the feedback channel. This is the handler persistence gap flagged in Phase 1 and Phase 8 learnings.

The fix: add an explicit `floor-manager-feedback` broadcast in the `task:queue-status` EventBus handler inside `setupEventHandlers()` so analytics-agent (and all virtual agent) task completions are routed to the floor-manager feedback chain — not just budget-plan.

## Patch

### Change 1: Extend the `task:queue-status` handler to broadcast `floor-manager-feedback` for all agents (not just budget-plan)

File: `src/orchestrator/index.ts`

Find:
```typescript
    // Queue-status events — broadcast to dashboard via Supabase realtime.
    // NOTE: Do NOT feed these back into improvementEngine — that creates an infinite loop.
    this.eventBus.on('task:queue-status', (data) => {
      const { taskId, floorId, agentId, payload } = data;
      broadcastFloorEvent(floorId, 'task:queue-status', {
        task_id: payload.task_id,
        status: payload.status,
        timestamp: payload.timestamp,
        agentId,
        taskId,
      }).catch(() => {});
    });
```

Replace with:
```typescript
    // Queue-status events — broadcast to dashboard AND floor-manager-feedback channel.
    // Phase 1 + Phase 8 learning: ALL virtual agent completions must be routed to the
    // floor-manager feedback channel so verification loops resolve without retry exhaustion.
    // This covers analytics-agent Phase 9 tasks (performance-review, ad-optimization,
    // growth-report) and any other virtual agent tasks going forward.
    // NOTE: Do NOT feed these back into improvementEngine — that creates an infinite loop.
    this.eventBus.on('task:queue-status', (data) => {
      const { taskId, floorId, agentId, payload } = data;

      // 1. Broadcast to dashboard (always)
      broadcastFloorEvent(floorId, 'task:queue-status', {
        task_id: payload.task_id,
        status: payload.status,
        timestamp: payload.timestamp,
        agentId,
        taskId,
      }).catch(() => {});

      // 2. Broadcast to floor-manager-feedback channel so FM verification loops
      //    can observe all task state transitions without polling.
      //    Required by Phase 1 + Phase 8 learnings for handler persistence.
      const task = this.taskManager.getTask(taskId);
      if (task) {
        broadcastFloorEvent(floorId, 'floor-manager-feedback', {
          channel: 'floor-manager-feedback',
          task_id: payload.task_id,
          current_queue_state: payload.status === 'complete' ? 'completed'
            : payload.status === 'failed' ? 'failed'
            : 'pending',
          execution_outcome: payload.status === 'complete' ? 'success'
            : payload.status === 'failed' ? 'failure'
            : 'pending',
          taskType: task.taskType,
          agentId,
          phaseNumber: task.phaseNumber,
          timestamp: payload.timestamp,
          result_summary: payload.result_summary,
        }).catch(() => {});
      }
    });
```

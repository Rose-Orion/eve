## Diagnosis

This is an investigation task. I need to: (1) check what brand state data exists in the database/floors for LevelUp Quest, (2) determine if there's a theme injection mechanism, and (3) if outputs exist but aren't wired into the UI theme, generate the patch.

Based on the code, `extractBrandState()` in `src/orchestrator/index.ts` parses the `brand-visual-system.md` deliverable and stores structured data (primaryColor, secondaryColor, palette, headingFont, bodyFont, logoDescription) into `floor.brandState` as a JSON string. The `Floor.brandState` field in the DB holds this.

However, looking at the `Floor` type, `brandState` is typed as `BrandState` (a string enum: `'pre-foundation' | 'foundation-review' | 'foundation-approved' | 'brand-revision'`), but the extraction code is **overwriting it with a JSON object** — a type collision that means once brand visuals are extracted, the enum value is lost and the field becomes a JSON blob.

More critically: **there is no mechanism that reads the extracted brand colors/typography from the floor's brandState and injects them into any CSS variables, theme config, or floor UI theme.** The data is stored in Supabase but never surfaces to the floor's UI layer.

Let me trace the full picture and generate the patch.

  fastify.get<{ Params: { id: string } }>('/floors/:id/theme', async (req, reply) => {
    const floor = orchestrator.getFloor(req.params.id);
    if (!floor) return reply.status(404).send({ error: 'Floor not found' });

    if (!floor.themeConfig) {
      return reply.status(204).send(); // No theme extracted yet
    }

    // Return theme as CSS variables + raw config for flexibility
    const theme = floor.themeConfig;
    const cssVars: Record<string, string> = {};
    if (theme.primaryColor) cssVars['--brand-primary'] = theme.primaryColor;
    if (theme.secondaryColor) cssVars['--brand-secondary'] = theme.secondaryColor;
    if (theme.accentColor) cssVars['--brand-accent'] = theme.accentColor;
    if (theme.backgroundColor) cssVars['--brand-bg'] = theme.backgroundColor;
    if (theme.textColor) cssVars['--brand-text'] = theme.textColor;
    if (theme.headingFont) cssVars['--brand-heading-font'] = theme.headingFont;
    if (theme.bodyFont) cssVars['--brand-body-font'] = theme.bodyFont;

    // Build palette CSS vars
    if (theme.palette?.length) {
      theme.palette.forEach((swatch, i) => {
        const slug = swatch.name.toLowerCase().replace(/\s+/g, '-');

        cssVars[`--brand-palette-${i}`] = swatch.hex;
      });
    }

    return reply.send({
      theme,
      cssVariables: cssVars,
      googleFontsUrl: buildGoogleFontsUrl(theme.headingFont, theme.bodyFont),

/**
 * Build a Google Fonts embed URL for the extracted heading/body fonts.
 * Returns null if both fonts are system fonts or undefined.
 */
function buildGoogleFontsUrl(headingFont?: string, bodyFont?: string): string | null {
  const SYSTEM_FONTS = new Set(['system-ui', 'sans-serif', 'serif', 'monospace', 'Arial', 'Georgia', 'Times New Roman']);
  const families: string[] = [];

  if (headingFont && !SYSTEM_FONTS.has(headingFont)) {
    families.push(headingFont.replace(/\s+/g, '+') + ':wght@400;600;700');
  }
  if (bodyFont && bodyFont !== headingFont && !SYSTEM_FONTS.has(bodyFont)) {
    families.push(bodyFont.replace(/\s+/g, '+') + ':wght@400;500');
  }

  if (families.length === 0) return null;
  return `https://fonts.googleapis.com/css2?${families.map(f => `family=${f}`).join('&')}&display=swap`;
}

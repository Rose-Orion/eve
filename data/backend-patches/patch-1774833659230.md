## Diagnosis

The chat history is stored only in a local JSON file (`data/chats/<floorId>.json`) with no Supabase persistence. The `saveChatMessage` and `loadChatMessages` functions exist in `supabase.ts` but are never called — `sendChatMessage` calls the local `persistChat`/`loadChatHistory` file-based methods instead. This means: (a) messages are never written to `floor_chat_messages` in Supabase, (b) session resume reads from a local file that won't survive server restarts on hosted environments, (c) there are no TTL/cleanup issues but the architecture is wrong — DB should be the source of truth. The fix is to make `sendChatMessage` write each message to Supabase immediately (before broadcasting), and make `loadChatHistory` read from Supabase first (falling back to the local file for migration compatibility).

## Patch

### Change 1: Replace file-based persistChat/loadChatHistory with Supabase-backed versions

File: `src/orchestrator/index.ts`

Find:
```typescript
  private chatDir = join(process.cwd(), 'data', 'chats');

  private async persistChat(floorId: string): Promise<void> {
    try {
      await mkdir(this.chatDir, { recursive: true });
      const history = this.chatHistories.get(floorId) ?? [];
      await writeFile(join(this.chatDir, `${floorId}.json`), JSON.stringify(history), 'utf-8');
    } catch { /* non-critical */ }
  }

  private async loadChatHistory(floorId: string): Promise<void> {
    if (this.chatHistories.has(floorId)) return;
    try {
      const data = await readFile(join(this.chatDir, `${floorId}.json`), 'utf-8');
      const history = JSON.parse(data);
      if (Array.isArray(history)) this.chatHistories.set(floorId, history);
    } catch { /* file doesn't exist yet */ }
  }
```

Replace with:
```typescript
  private chatDir = join(process.cwd(), 'data', 'chats');

  /**
   * Persist a single chat message to Supabase (primary store).
   * Falls back to local JSON file if Supabase is unavailable.
   */
  private async persistChatMessage(
    floorId: string,
    role: 'user' | 'assistant',
    content: string,
    sender: string,
  ): Promise<void> {
    // Primary: write to Supabase
    try {
      await saveChatMessage({ floorId, role, content, sender, timestamp: new Date() });
    } catch {
      // Secondary: fall back to local file
      try {
        await mkdir(this.chatDir, { recursive: true });
        const history = this.chatHistories.get(floorId) ?? [];
        await writeFile(join(this.chatDir, `${floorId}.json`), JSON.stringify(history), 'utf-8');
      } catch { /* non-critical */ }
    }
  }

  /**
   * Load chat history for a floor.
   * Primary source: Supabase floor_chat_messages table (ordered by timestamp).
   * Fallback: local JSON file (for migration from old file-based storage).
   */
  private async loadChatHistory(floorId: string): Promise<void> {
    if (this.chatHistories.has(floorId)) return;

    // Try Supabase first
    try {
      const dbMessages = await loadChatMessages(floorId);
      if (dbMessages.length > 0) {
        this.chatHistories.set(
          floorId,
          dbMessages.map(m => ({
            role: m.role,
            content: m.content,
            timestamp: m.timestamp ?? new Date(),
          })),
        );
        return;
      }
    } catch { /* Supabase unavailable — fall through to file */ }

    // Fall back to local file (migration path for existing deployments)
    try {
      const data = await readFile(join(this.chatDir, `${floorId}.json`), 'utf-8');
      const history = JSON.parse(data);
      if (Array.isArray(history) && history.length > 0) {
        this.chatHistories.set(floorId, history);
        // Migrate local history into Supabase so future loads use DB
        for (const msg of history) {
          const role = msg.role === 'assistant' ? 'assistant' : 'user';
          const sender = role === 'assistant' ? 'floor-manager' : 'owner';
          await saveChatMessage({
            floorId,
            role,
            content: msg.content,
            sender,
            timestamp: msg.timestamp ? new Date(msg.timestamp) : new Date(),
          }).catch(() => {});
        }
        console.log(`[Chat] Migrated ${history.length} messages from file to Supabase for floor ${floorId}`);
      }
    } catch { /* file doesn't exist yet — start fresh */ }
  }
```

## Verification

1. **Message write path**: Send a chat message via the API, then query Supabase directly:
   ```sql
   SELECT * FROM floor_chat_messages WHERE floor_id = '<your-floor-id>' ORDER BY timestamp;
   ```
   You should see both the user message and assistant response with correct `role`, `content`, `sender`, and `timestamp` values.

2. **Session resume**: Restart the server process, then call `GET /api/floors/:id/chat` (or equivalent). The history should be loaded from Supabase, not from a local file. Confirm `loadChatHistory` takes the Supabase branch (add a temp `console.log` if needed).

3. **Migration**: If a `data/chats/<floorId>.json` file exists from before this fix, on first load the messages will be migrated to Supabase and the log will show `[Chat] Migrated N messages from file to Supabase for floor <id>`.

4. **No accidental deletion**: Grep the codebase for `clearChatMessages` — it should only be called from `clearFloorChat`, which is only invoked by explicit owner action (no background jobs).

5. **Sender field**: Verify `sender` column in `floor_chat_messages` is populated: `owner` for user messages, `floor-manager` for assistant responses.

## Rollback

To revert to the file-based approach if something breaks:

1. Restore the three original methods (`persistChat`, `loadChatHistory`, and the `sendChatMessage` body) from this diff.
2. The local JSON files in `data/chats/` are untouched by this change, so no data is lost on rollback.
3. If you need to revert only the Supabase writes while keeping the new structure, set `saveChatMessage` to be a no-op temporarily by returning early in `persistChatMessage` before the `saveChatMessage` call.

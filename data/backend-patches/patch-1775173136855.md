## Diagnosis

The issue is a repeated failure of the `product-catalog` task assigned to `commerce-agent` on Glow Lab's Phase 4. I need to: (1) inspect the commerce-agent's deployment status and health, (2) check the task schema/prompt for the `product-catalog` task as seeded in `seedNextPhaseTasks` Phase 4, and (3) identify any input validation failure or misconfiguration.

Reading the Phase 4 seeding logic in `seedNextPhaseTasks`, the `product-catalog` task is only created when `isEcommerce` is true (`floor.config.businessType === 'ecommerce'`). The prompt lacks key required fields (ingredient lists, structured pricing schema, imagery references) that `commerce-agent` expects for a beauty/skincare brand like Glow Lab. Additionally, there is no `dependsOn` array specified, which means it dispatches in parallel without waiting for the brand visual system — causing the agent to produce generic output without brand context. The prompt also doesn't specify the output schema that the agent needs.

The diagnostic route needs to be added to expose the task state, and the product-catalog prompt needs to be enriched with the required fields for a beauty brand.

## Diagnosis

The issue has three parts: (1) we need to confirm the `task:completed` event handler is wired and add ERROR-level logging to its dependency-resolution and enqueue steps so misfires become visible in logs; (2) we need a diagnostic method to check whether `brand-details` is missing from Quest Kids Phase 1 and re-enqueue it if absent; (3) a manual re-enqueue endpoint should exist for `brand-details`. Looking at the code, `setupEventHandlers()` has a `task:completed` handler but it has **no error logging** when `dependencyGraph.getReadyTasks()` returns results or when `taskManager.transition()` fails — silent failures mean downstream tasks get skipped with no trace. Additionally, Phase 3 (Foundation Sprint) only seeds `brand-options`, `business-strategy`, and `budget-plan` — there is no `brand-details` task defined in any phase's seed template, which means it can never exist unless added.

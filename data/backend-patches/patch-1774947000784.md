## Diagnosis

The "No data yet" in the Agent Performance panel is caused by a missing metrics write path: `improvementEngine.recordTaskCompletion()` is called in `dispatchVirtual()` after a task succeeds, but the Dashboard API query for agent performance reads from a different source than what `recordTaskCompletion` writes to — specifically, the `getPerformance()` data used in `floorManagerReview()` is never surfaced through the dashboard API route for the Agent Performance panel. The panel likely calls a route that queries Supabase `task_metrics` or similar, but completed tasks only update the in-memory `ImprovementEngine` state and the `floors.spent_cents` column, with no write to a `task_metrics` or `agent_performance` table.

Additionally, the $4 figure for 10 tasks needs verification: at ~$0.40/task average that's plausible for Haiku/Sonnet calls, but we need to confirm `saveCostEvent` is actually writing to Supabase (it's called via `persistWithRetry` in `dispatchVirtual` only when `result.costCents > 0`).

Let me trace the full pipeline and provide targeted fixes.

    const byAgent = new Map<string, {

      firstTryCount: number;

      taskTypes: string[];

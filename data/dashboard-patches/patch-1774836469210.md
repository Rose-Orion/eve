## Diagnosis

Three independent changes required: (1) the chat panel needs to be repositioned to appear to the right of the floor content by restructuring the floor view layout into a flex row; (2) chat history is only stored in-memory (`chatPanelMessages`) and lost on refresh — needs localStorage persistence; (3) the brand visual system tokens exist in the CSS `:root` (`--brand-primary: #4A3AFF`, `--brand-secondary: #FFB833`, `--brand-accent: #2ECC71`) but are not being applied to floor UI components — buttons, cards, typography, progress bars, and navigation all still use the generic `--indigo`/`--violet` variables instead of the brand tokens.

### Change 1a: Persist chat history to localStorage on every message sent

File: `public/app.js`

Find:
```javascript
// In-memory message store keyed by floorId
const chatPanelMessages = {};
```

Replace with:
```javascript
// Persistent message store keyed by floorId — backed by localStorage
const CHAT_HISTORY_KEY = 'eve-chat-history';

function loadChatHistory() {
  try {
    return JSON.parse(localStorage.getItem(CHAT_HISTORY_KEY) || '{}');
  } catch { return {}; }
}

function saveChatHistory(store) {
  try {
    // Keep last 100 messages per floor to avoid unbounded growth
    const trimmed = {};
    for (const [fid, msgs] of Object.entries(store)) {
      trimmed[fid] = Array.isArray(msgs) ? msgs.slice(-100) : [];
    }
    localStorage.setItem(CHAT_HISTORY_KEY, JSON.stringify(trimmed));
  } catch {}
}

// In-memory message store — initialised from localStorage so history survives refreshes
const chatPanelMessages = loadChatHistory();
```

## Verification

**Change 1 (Chat history persistence):**
1. Open a floor, send 3 messages in the Floor Manager chat panel
2. Hard-refresh the page (Cmd+Shift+R / Ctrl+Shift+R)
3. Navigate back to the same floor — all 3 messages must be visible in the panel
4. Check `localStorage` in DevTools → Application → `eve-chat-history` — should contain the messages as JSON

**Change 2 (Chat panel right-side positioning):**
1. Navigate to any floor's building view
2. The chat panel must appear anchored to the right edge of the screen
3. The floor content (progress, budget, phases) must fill the left/center area
4. When the panel is minimized to the slim rail, floor content expands to use the freed space
5. On mobile (< 600px), the panel overlays full-width (no content shift)

**Change 3 (Brand visual system):**
1. Open the Tower — the aurora background should show a deeper Electric Indigo bloom top-left with a subtle Quest Gold tint top-right
2. The "New Floor" card hover should glow with brand indigo (#4A3AFF)
3. Page titles (e.g. "The Tower", floor names) should render with a white-to-gold gradient text
4. Section labels (e.g. "ACTIVITY FEED", "BUILD PROGRESS") should appear in Quest Gold (#FFB833)
5. All progress bars must use the brand gradient (indigo → 7B5FFF) with a subtle glow
6. Primary buttons ("Continue →", "Build It") must use the brand gradient and glow
7. Active step dots in the stepper must glow brand indigo
8. The Floor Manager chat send button must use the brand gradient

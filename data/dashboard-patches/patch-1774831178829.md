## Diagnosis

The current `viewChat` function renders a full-page view that replaces the main floor content entirely. We need to replace this with a persistent right-side panel architecture: a fixed panel anchored to the viewport right edge, with a toggle to collapse it to a slim sidebar rail, and main content that shifts left to accommodate it (not overlapped). The panel must preserve scroll position and message history across expand/collapse cycles.

This requires: (1) a persistent panel DOM element injected once at `init()` time (not re-created on `render()`), (2) CSS layout that shifts `#app` via a right margin/padding when the panel is expanded, (3) the `floor-chat` route now opens the panel instead of navigating away, and (4) the toggle and chat state living outside the router so they survive view changes.

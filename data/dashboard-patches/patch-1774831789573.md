## Diagnosis

The `renderTopNav()` function always renders the notification bell (🔔) in the non-back-button variant (the Tower-level nav). The Tower view calls `renderTopNav()` with no arguments, so it hits the default branch that includes the bell. Floor views call `renderTopNav({ back: true, ... })`, which renders the back-button nav — but that nav has no bell at all currently. The fix is: (1) suppress the bell in the Tower nav, and (2) add the bell to the back-button nav variant when on a Floor-level view.

## Patch

### Change 1: Remove bell from Tower nav, add bell to Floor-level back navs

The `renderTopNav` function needs a new option — `showNotifications` — that floor views can opt into. Tower calls `renderTopNav()` with no args (bell suppressed by default). Floor views pass `showNotifications: true`.

**File:** `public/app.js`

**Find:**
```javascript
function renderTopNav(opts = {}) {
  const pendingCount = state.approvals.filter(a => a.status === 'pending').length;
  const hasPending = pendingCount > 0;

  if (opts.back) {
    const backOnclick = opts.backOnclick || 'router.back()';
    return `
    <nav class="topnav">
      <button class="back-btn" onclick="${backOnclick}">
        <svg viewBox="0 0 16 16" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round">
          <path d="M10 12L6 8l4-4"/>
        </svg>
        ${opts.backLabel || 'Back'}
      </button>
      <span style="font-size:14px;font-weight:600;color:var(--text-2);letter-spacing:0.3px">${opts.title || ''}</span>
      <div style="width:80px"></div>

  }

  return `
  <nav class="topnav">
    <a class="topnav-logo" onclick="router.go('tower')" href="javascript:void(0)">
      <span class="star">✦</span>
      <span>EVE</span>
    </a>
    <div class="topnav-actions">
      <div class="relative">
        <button class="icon-btn" title="Notifications" onclick="router.go('notifications')">🔔</button>

      </div>
    </div>

}
```

**Replace with:**
```javascript
function renderTopNav(opts = {}) {
  const pendingCount = state.approvals.filter(a => a.status === 'pending').length;
  const hasPending = pendingCount > 0;

  if (opts.back) {
    const backOnclick = opts.backOnclick || 'router.back()';
    const notifHtml = opts.showNotifications ? `
      <div class="relative">
        <button class="icon-btn" title="Notifications" onclick="router.go('notifications')">🔔</button>
        ${hasPending ? '<div class="notif-badge"></div>' : ''}
      </div>` : '<div style="width:40px"></div>';
    return `
    <nav class="topnav">
      <button class="back-btn" onclick="${backOnclick}">
        <svg viewBox="0 0 16 16" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round">
          <path d="M10 12L6 8l4-4"/>
        </svg>
        ${opts.backLabel || 'Back'}
      </button>
      <span style="font-size:14px;font-weight:600;color:var(--text-2);letter-spacing:0.3px">${opts.title || ''}</span>
      ${notifHtml}
    </nav>`;
  }

  // Tower-level nav: no notification bell
  return `
  <nav class="topnav">
    <a class="topnav-logo" onclick="router.go('tower')" href="javascript:void(0)">
      <span class="star">✦</span>
      <span>EVE</span>
    </a>
    <div style="width:40px"></div>
  </nav>`;
}
```

## Verification

1. **Tower view** — navigate to the Tower (`/` or tap EVE logo). Confirm the top nav shows only the EVE logo on the left. The 🔔 bell should be **absent**.
2. **Floor building view** — tap any floor card that is still building. Confirm the top nav shows the back button, floor name centered, and the 🔔 bell on the right. If there are pending approvals, the red dot badge should appear on it.
3. **Floor live view** — tap a live floor. Same check: bell present with badge if pending approvals exist.
4. **All other back-nav views** (gate1, gate2, gate3, weekly, activity, etc.) — these use `renderTopNav({ back: true, ... })` without `showNotifications`, so they will render a blank right-side spacer (`width:40px`). The bell will not appear there — which is the conservative safe default and consistent with the owner's intent to only show it at the floor level.
5. **Notification badge** — create a pending approval scenario and confirm the red dot appears on the floor-view bell but never on the Tower nav.

## Diagnosis

The floor view currently renders a generic UI regardless of which floor is being viewed. The brand visual system (colors, typography, logo, name) stored in `state.gate1BrandData[floorId]` and `localFloorState[floorId].brand` is only partially surfaced in the confirmed-brand card widget, but the floor page's background, header, navigation accent, section labels, buttons, and overall visual feel remain default EVE theme. We need to scope brand theming CSS variables to the floor view container and inject them dynamically from the selected brand's data.

The approach:
1. Inject a `<style>` tag with scoped CSS variables into the floor view when a brand is confirmed (`brandApproved === true`)
2. Override the top nav accent, page title color, buttons, progress bars, section labels, and background glow to use brand colors
3. Display the brand logo (emoji + name) prominently in the floor nav/header area
4. Keep all changes scoped to `.floor-branded` wrapper — no other views affected

// Returns a <style> block that scopes brand colors/typography to .floor-branded
// Only called when brandApproved === true. Scoped so it never bleeds to other views.
function getBrandThemeCSS(brand) {
  if (!brand) return '';
  const [c1, c2, c3 = c1] = brand.colors;
  // Derive a readable "on-brand" text color (white for dark primaries, dark for light)
  const isLight = (hex) => {
    const r = parseInt(hex.slice(1,3),16);
    const g = parseInt(hex.slice(3,5),16);
    const b = parseInt(hex.slice(5,7),16);
    return (r*299 + g*587 + b*114) / 1000 > 140;
  };
  const onBrand = isLight(c1) ? '#0a0a1e' : '#ffffff';
  const onBrand2 = isLight(c2) ? '#0a0a1e' : '#ffffff';

  // Build font stack — if brand has a concept implying a style, hint at font
  const concept = (brand.concept || '').toLowerCase();
  const fontStack = concept.includes('modern') || concept.includes('tech')
    ? "'Inter', 'SF Pro Display', system-ui, sans-serif"
    : concept.includes('bold') || concept.includes('street') || concept.includes('quest') || concept.includes('game')
    ? "'Inter', 'SF Pro Display', system-ui, sans-serif"
    : "'Inter', system-ui, sans-serif";

  return `
<style id="floor-brand-theme">
/* ── Brand Theme: ${brand.brandName} — scoped to .floor-branded ── */
.floor-branded {
  --brand-primary:   ${c1};
  --brand-secondary: ${c2};
  --brand-accent:    ${c3};
  --brand-on-primary: ${onBrand};
  --brand-on-secondary: ${onBrand2};
  --brand-gradient: linear-gradient(135deg, ${c1}, ${c2});
  --brand-gradient-soft: linear-gradient(135deg, ${c1}22, ${c2}11);
  --brand-border: ${c1}55;
  --brand-glow: 0 0 40px ${c1}33;
  --brand-font: ${fontStack};
}

.floor-branded .floor-brand-ambient {
  background: radial-gradient(ellipse 90% 50% at 50% -10%, ${c1}1a 0%, transparent 65%),
              radial-gradient(ellipse 60% 40% at 90% 80%, ${c2}0f 0%, transparent 55%);
}

/* Page title — uses brand primary */
.floor-branded .page-title {
  color: ${c1} !important;
  font-family: var(--brand-font);
  letter-spacing: -0.8px;
}

/* Section labels — tinted with brand secondary */

  color: ${c2}cc;
  letter-spacing: 1.2px;
}

/* Primary button — brand gradient */

  background: var(--brand-gradient) !important;
  box-shadow: 0 4px 20px ${c1}44 !important;
}
.floor-branded .btn-primary:hover {
  box-shadow: 0 6px 30px ${c1}66 !important;
}

/* Success button — brand accent tint */

  box-shadow: 0 4px 20px ${c2}44 !important;
}

/* Progress bars — brand gradient */
.floor-branded .progress-bar-fill {
  background: var(--brand-gradient) !important;
}
.floor-branded .floor-progress-bar {
  background: var(--brand-gradient) !important;
}

/* Phase item active state — brand tinted */
.floor-branded .phase-item.active {
  border-color: ${c1}55 !important;
  background: ${c1}0f !important;
}
.floor-branded .phase-item.active .phase-status {
  color: ${c1} !important;
  background: ${c1}1a !important;
}

/* Phase mini-bar — brand gradient */

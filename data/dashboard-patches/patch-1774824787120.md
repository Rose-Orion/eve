## Diagnosis

The 'Review →' button inside the `viewFloorBuilding` function is generated by the `brandReady` branch's inline `onclick` handler. When the `gateView` fallback resolves to `'floor'` (the unknown gate case), the button navigates to `{ view: 'floor', params: { id: ... } }` — which is exactly the current page, creating a self-loop. For floor `b090c7ea-ef9c-4bb3-abdf-62897e1224e8`, the `pendingGate` approval either has an unrecognized `phaseNumber` or is `undefined`, causing `gateView` to fall back to `'floor'` instead of the correct gate view.

There are **two locations** in `viewFloorBuilding` where this fallback exists — the pulsing review banner click handler, and the bottom `btn-row` button — both must be fixed. The correct fallback for an unrecognized gate should be `'gate1'` (the most likely review destination for an in-progress floor), not `'floor'`.

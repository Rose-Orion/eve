## Diagnosis

The brand-visual-system deliverable has defined SideQuest's design tokens (Electric Indigo `#4A3AFF`, Quest Gold `#FFB833`, Emerald `#2ECC71`, Inter typeface, 16px border radius, specific shadow values). The current `styles.css` already has these tokens partially wired into `:root` CSS variables, but several component-level styles still reference old hardcoded values (e.g., `#6366f1` indigo, `#8b5cf6` violet, generic gradients) instead of the brand token variables. The fix is a targeted token-mapping pass: update all component-level color references to use `var(--brand-primary)`, `var(--brand-secondary)`, `var(--brand-accent)`, and related variables, without changing any layout or structure.

---

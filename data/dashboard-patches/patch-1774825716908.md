## Diagnosis

The "content production ready for review" button in `viewFloorBuilding` routes based on `pendingGate?.phaseNumber`. Content Production is backend phase 5, but the gate routing logic only maps phase numbers 3 → `gate1`, 6 → `gate2`, and 8 → `gate3`. Phase 5 falls through to the default `'gate1'` fallback — pointing to the brand/foundation review screen instead of the correct content output review screen (`gate3`). The fix is to map phase number 5 to `gate3` (the content/ad review screen, which is the appropriate output review view for Content Production).

Wait — let me re-read the phase mapping carefully before finalising:

- Phase 3 = Foundation Sprint → `gate1` ✅
- Phase 6 = Staging & QA → `gate2` ✅  
- Phase 8 = Ad Campaign Activation → `gate3` ✅
- Phase 5 = Content Production → currently falls through to `gate1` ❌

The content production review should route to `gate3` (the ad/content review screen, which handles creative review). This fix appears in **three places** in `viewFloorBuilding`: the pulsing banner `onclick`, the bottom `btn-row` button `onclick`, and the `gateView` variable used for the banner's `router.go` call.

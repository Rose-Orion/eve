## Diagnosis

The `viewGate1` function currently uses `backOnclick: \`router.go('floor', {id:'${floor.id}'})\`` and `backLabel: floor.name` — so the destination IS correct (it navigates back to the floor detail view), but the label `floor.name` (which renders as "LevelUp Quest") is ambiguous. A button labeled just "LevelUp Quest" doesn't communicate *where* it goes — the owner doesn't know if pressing it takes them to the floor overview, the tower, or somewhere else. The label needs to clearly indicate the destination.

Additionally, I'll audit all three gate views to ensure consistent back-button behavior: all gates should return to the floor detail (`floor` view), not the tower.

**Gate views audit:**
- `viewGate1`: `backOnclick → router.go('floor', {id})` ✅ correct destination, label is `floor.name` ⚠️ ambiguous
- `viewGate1Voice`: uses `router.back()` (default) ⚠️ relies on history stack — could go anywhere
- `viewGate1Strategy`: uses `router.back()` (default) ⚠️ same issue
- `viewGate2`: `backOnclick → router.go('floor', {id})` ✅ correct, label is `floor.name` ⚠️ same ambiguity
- `viewGate3`: `backOnclick → router.go('floor', {id})` ✅ correct, label is `floor.name` ⚠️ same ambiguity

All gate back buttons should: (1) explicitly navigate to `floor` view (not rely on history), (2) use a label that clearly says "← Back to [FloorName] Floor" so the owner knows exactly where they'll land.
